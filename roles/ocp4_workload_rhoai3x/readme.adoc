= ocp4_workload_rhoai3x — Red Hat OpenShift AI 3.x

== Overview

Provisions a full *Red Hat OpenShift AI (RHOAI) 3.3* GPU demo environment on OpenShift 4 (AWS).
Runs as Stage 2 of the `openshift-workloads-rhoai` config, after NFD, MinIO, and htpasswd auth are already deployed by Stage 1.

=== Provisioning Order

[source]
----
nfd_instance → gpu_machineset → nvidia_gpu_operator → rhoai_operator
  → datasciencecluster → mlflow_instance → pipeline_app
  → model_deployment → gen_ai_playground
----

=== Prerequisites (Stage 1)

The following workloads must already be deployed before running this role:

* `ocp4_workload_authentication_htpasswd` — admin user + student users
* `ocp4_workload_nfd` — Node Feature Discovery operator (labels GPU nodes automatically)
* `ocp4_workload_minio` — S3-compatible storage for model artifacts and pipeline logs

=== What This Role Deploys

. *NodeFeatureDiscovery instance* — Creates the `NodeFeatureDiscovery` CR in `openshift-nfd`, starting the NFD daemonset that labels GPU nodes with PCI feature labels needed by the GPU Operator.

. *GPU MachineSet* — Clones an existing worker MachineSet on AWS, switches the instance type to a GPU instance (`g6e.4xlarge` by default), and provisions a 200 GB gp3 root volume. Pre-seeds the NFD PCI label on the MachineSet template so new GPU nodes are recognized immediately. Waits for nodes to reach `Ready` status.

. *NVIDIA GPU Operator* — Installed via OLM from the `certified-operators` catalog (`stable` channel). Creates a `ClusterPolicy` with OCP Driver Toolkit enabled — the DTK automatically selects the correct NVIDIA driver version for the RHCOS kernel. Waits for `ClusterPolicy` to reach `ready` state and for `nvidia.com/gpu` to appear in node allocatable resources.

. *RHOAI Operator* — Installed via OLM from `redhat-operators` (`stable-3.x` channel, AllNamespaces mode). Waits for the CSV to succeed.

. *DataScienceCluster* — Enables: Dashboard, Workbenches, KServe (RawDeployment/Headless mode — no Service Mesh or Serverless required), AI Pipelines, LlamaStack Operator, MLflow Operator, and Model Registry. Patches `OdhDashboardConfig` to enable the Gen AI Studio Tech Preview feature flag, then restarts the dashboard.

. *MLflow instance* — Creates an `MLflow` CR in `redhat-ods-applications` (the MLflow operator's fixed target namespace). Uses SQLite on a PVC for metadata and MinIO from Stage 1 for artifact storage. Accessible at `https://data-science-gateway.apps.<cluster-domain>/mlflow`.

. *DataSciencePipelineApplication* — Pre-creates a shared AI Pipelines instance in the `rhoai-shared` project, backed by the MinIO instance from Stage 1.

. *Model deployment* — Creates a `ServingRuntime` and `InferenceService` via KServe. Defaults to `gpt-oss-20b` served with the RHAIIS vLLM CUDA image, which natively handles mxfp4 quantization via the Marlin kernel. Waits for the InferenceService to reach `Ready` status.

. *Gen AI Playground* — Creates a `LlamaStackDistribution` in `rhoai-shared`, wired to the deployed InferenceService, and backed by a ConfigMap with the full LlamaStack configuration (inference, vector I/O, agents, pipelines). Accessible from the RHOAI dashboard.

== Access

=== RHOAI Dashboard

RHOAI 3.3 uses an Ingress Gateway instead of a Route. The dashboard is not accessible via a Route in `redhat-ods-applications`.

[source,bash]
----
DOMAIN=$(oc get ingresses.config.openshift.io cluster -o jsonpath='{.spec.domain}')
echo "https://data-science-gateway.${DOMAIN}"
----

=== MLflow

Exposed via path-based routing under the same gateway. Also linked in the OCP console Application Menu → *OpenShift Self Managed Services → MLflow*.

[source,bash]
----
DOMAIN=$(oc get ingresses.config.openshift.io cluster -o jsonpath='{.spec.domain}')
echo "https://data-science-gateway.${DOMAIN}/mlflow"
----

To instrument workloads with MLflow tracking:

[source,bash]
----
export MLFLOW_TRACKING_TOKEN="$(oc whoami --show-token)"
export MLFLOW_TRACKING_URI="https://data-science-gateway.${DOMAIN}/mlflow"
export MLFLOW_WORKSPACE="<your-namespace>"
----

NOTE: The token is session-scoped. Refresh it with `oc whoami --show-token` each time.

== Verification

[source,bash]
----
# GPU node — check Ready and nvidia.com/gpu allocatable
oc get nodes -l nvidia.com/gpu.present=true
oc get node <gpu-node> -o jsonpath='{.status.allocatable.nvidia\.com/gpu}'

# NVIDIA GPU Operator
oc get csv -n nvidia-gpu-operator | grep gpu-operator
oc get clusterpolicy gpu-cluster-policy -o jsonpath='{.status.state}'

# RHOAI Operator
oc get csv -n redhat-ods-operator | grep rhods

# DataScienceCluster — check all Ready components
oc get datasciencecluster default-dsc -o jsonpath='{.status.conditions}' | \
  python3 -c "import sys,json; [print(c['type'],c['status']) for c in json.load(sys.stdin)]"

# Model serving
oc get inferenceservice -n rhoai-shared
oc get servingruntime -n rhoai-shared

# Gen AI Playground
oc get llamastackdistribution -n rhoai-shared

# MLflow
oc get mlflow -n redhat-ods-applications

# AI Pipelines
oc get datasciencepipelinesapplication -n rhoai-shared
----
