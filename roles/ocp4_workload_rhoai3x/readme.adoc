= ocp4_workload_rhoai3x ‚Äî Red Hat OpenShift AI 3.x

== Overview

Stage 2 workload role that installs and configures *Red Hat OpenShift AI (RHOAI) 3.3* on an OpenShift 4 cluster on AWS (Red Hat Demo Platform).

=== Prerequisites (Stage 1)

The following workloads must already be deployed before running this role:

* `ocp4_workload_authentication_htpasswd` ‚Äî admin user + student users
* `ocp4_workload_nfd` ‚Äî Node Feature Discovery (labels GPU nodes automatically)
* `ocp4_workload_minio` ‚Äî S3-compatible storage for model artifacts and pipeline logs

=== What This Role Does

. *GPU MachineSet* ‚Äî Clones an existing worker MachineSet and provisions GPU nodes (AWS `g6e.4xlarge` by default). Waits for nodes to reach `Ready` status before continuing.
. *NVIDIA GPU Operator* ‚Äî Installed via OLM from the `certified-operators` catalog. Waits for the `ClusterPolicy` to reach `ready` state.
. *RHOAI Operator* ‚Äî Installed via OLM from `redhat-operators`, channel `fast-3.3`. Waits for the CSV to succeed.
. *DataScienceCluster* ‚Äî Enables Dashboard, Workbenches, KServe (RawDeployment mode), and AI Pipelines. Waits for `Ready` phase.
. *DataSciencePipelineApplication* ‚Äî Pre-creates a shared AI Pipelines instance in the `rhoai-shared` project, backed by the MinIO instance from Stage 1.

== Verification

[source,bash]
----
# GPU nodes
oc get nodes -l node-role.kubernetes.io/gpu=

# NVIDIA GPU Operator
oc get csv -n nvidia-gpu-operator | grep gpu-operator
oc get clusterpolicy gpu-cluster-policy

# RHOAI Operator
oc get csv -n redhat-ods-operator | grep rhods
oc get pods -n redhat-ods-applications

# DataScienceCluster
oc get datasciencecluster default-dsc -o jsonpath='{.status.phase}'

# RHOAI Dashboard route
oc get route -n redhat-ods-applications | grep dashboard

# AI Pipelines
oc get datasciencepipelinesapplication -n rhoai-shared
----
